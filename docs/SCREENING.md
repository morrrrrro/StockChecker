# スクリーニング戦略書

## 概要

3つの独立したスクリーニング戦略と、マルチファクタースコアリングシステムにより、
定量的に「買い目」の銘柄を特定する。

## スクリーニング戦略

### Screen A: バリュー＋クオリティ（Greenblatt-Piotroski ハイブリッド）

Magic Formula（Joel Greenblatt）とPiotroski F-Scoreを組み合わせた戦略。
財務データに基づくため、最もファンダメンタルに裏付けられたスクリーン。

#### 条件

| 指標 | 算出方法 | 条件 |
|---|---|---|
| 益利回り | EBIT / EV | ユニバース上位20% |
| 投下資本利益率 | EBIT / (運転資本 + 固定資産) | ユニバース上位20% |
| Piotroski F-Score | 9項目の財務健全性スコア | 7以上 |

#### Piotroski F-Score 算出項目（各1点、合計9点）

**収益性（4点）:**
1. ROA > 0（当期純利益 / 総資産）
2. 営業キャッシュフロー > 0
3. ROAが前年比で改善
4. 営業CF / 総資産 > 当期純利益 / 総資産（アクルーアル品質）

**財務レバレッジ・流動性（3点）:**
5. 長期負債 / 総資産が前年比で減少
6. 流動比率が前年比で改善
7. 当期に新株発行なし

**営業効率（2点）:**
8. 粗利率が前年比で改善
9. 資産回転率（売上高 / 総資産）が前年比で改善

#### データソース
- 財務データ: EDINET API（有価証券報告書のXBRLデータ）
- 補助: yfinance `.info`（PER、PBR、ROE等の速報値）

### Screen B: モメンタムブレイクアウト

価格モメンタムと出来高を組み合わせた短中期のブレイクアウト戦略。

#### 条件

| 指標 | 算出方法 | 条件 |
|---|---|---|
| 6ヶ月リターン | (現在値 - 6ヶ月前) / 6ヶ月前 | ユニバース上位20% |
| トレンド確認 | 株価 vs 200日移動平均線 | 200日線の上 |
| 出来高確認 | 当日出来高 / 20日平均出来高 | 1.5倍以上 |
| RSI(14) | 相対力指数 | 30〜70の健全レンジ |

#### 補足
- RSI < 30でもモメンタム上位なら矛盾（通常はあり得ない）→ 除外
- RSI > 70は過熱圏のため除外（追いかけ買い防止）
- ゴールデンクロス（25日線が75日線を上抜け）は追加ポイント

### Screen C: 配当バリュー

高配当かつ割安で財務健全な銘柄を抽出する長期投資向けスクリーン。

#### 条件

| 指標 | 算出方法 | 条件 |
|---|---|---|
| 配当利回り | 年間配当 / 株価 | 3%以上 |
| PBR | 株価 / 1株純資産 | 1.2倍以下 |
| F-Score | Piotroski F-Score | 6以上 |
| 高値からの下落率 | (現在値 - 52週高値) / 52週高値 | -15%以内 |

#### 補足
- 減配リスクの高い銘柄を除外するため、F-Score閾値を設定
- 下落率制限により、構造的に問題のある銘柄を排除

## マルチファクタースコアリング

### 算出方法

各ファクターをユニバース内のパーセンタイルランク（0-100）に変換し、加重平均で合成する。
パーセンタイルはz-scoreより外れ値に頑健。

```
Composite Score = 0.35 × Value + 0.30 × Quality + 0.25 × Momentum + 0.10 × Risk
```

### サブスコア算出

```
Value Score    = avg(percentile(益利回り), percentile(1/PBR), percentile(配当利回り))
Quality Score  = avg(percentile(ROE), percentile(FCFマージン), percentile(F-Score))
Momentum Score = 0.5 × percentile(6M_return) + 0.3 × percentile(RS_rank) + 0.2 × above_200dma
Risk Score     = avg(percentile(1/ボラティリティ), percentile(1/最大DD), percentile(流動性))
```

- `above_200dma`: 200日線の上なら100、下なら0
- `RS_rank`: ユニバース内の相対強度ランキング（6ヶ月リターンの順位）

### 階層型フィルター（矛盾シグナル対策）

マルチファクタースコアだけでなく、階層的にゲートを設けて品質を担保する。

1. **品質ゲート**: F-Score ≦ 3 → 無条件除外（財務危険銘柄）
2. **バリューランク**: 通過銘柄をバリュー指標でランク付け
3. **モメンタム確認**: 200日移動平均線の上にある銘柄のみ残す
4. **例外ルール**: RSI < 30（売られすぎ）だがファンダメンタルズ悪化中 → スキップ

## 必須リスク指標

買いシグナルには以下のリスク指標を常に併記する。

### ボラティリティ

| 指標 | 算出方法 | 用途 |
|---|---|---|
| ATR(14日) | 14日間の真の範囲の平均 / 株価 | ポジションサイジング（リスク1% = 投資額 × 1% / ATR%） |
| ヒストリカルボラティリティ(20日) | 日次リターンの標準偏差を年率化 | 市場平均の2倍超は警告フラグ |

### ドローダウン

| 指標 | 算出方法 | 用途 |
|---|---|---|
| 52週高値からの下落率 | (現在値 - 52週高値) / 52週高値 | 損失ポテンシャルの参考 |
| 最大ドローダウン(6ヶ月) | 直近6ヶ月のピーク→トラフの最大下落率 | 30%超は警告フラグ |

### 流動性

| 指標 | 算出方法 | 閾値 |
|---|---|---|
| 20日平均出来高 | 直近20日の出来高平均 | 20万株 or 5000万円以上 |
| 出来高比率 | 当日出来高 / 20日平均出来高 | 2.0超で異常活動フラグ |

### リスク/リワード

| 指標 | 算出方法 | 条件 |
|---|---|---|
| サポート距離 | 直近スイングローまたは200日線（近い方） | 参考値 |
| レジスタンス距離 | 直近高値または前回の抵抗線 | 参考値 |
| R:R比率 | (現在値 - サポート) / (レジスタンス - 現在値) | 0.5未満のみ表示推奨 |
| セクターβ | 60日間の対セクター相関 | 参考値 |

## シグナルライフサイクル管理

スクリーニングで検出されたシグナルは、日次で状態を追跡する。

### 状態遷移

```
[未検出] → 新規 → 強化中 → 安定 → 減衰中 → [消失]
```

| 状態 | 条件 | 表示 |
|---|---|---|
| 新規 | シグナルが直近1-2営業日で初めて検出 | 黄バッジ |
| 強化中 | 追加ファクターが確認された（例: 出来高増加、RSI改善） | 緑バッジ |
| 安定 | シグナル継続中、新たな変化なし | 青バッジ |
| 減衰中 | 条件を満たさなくなりつつある（例: RSIが70に接近） | 灰バッジ |
| 消失 | 条件を満たさなくなった | 表示から削除、履歴に記録 |

### 追跡データ

```
signal_lifecycle テーブル:
  - ticker: 銘柄コード
  - signal_type: シグナル種別（screen_a / screen_b / screen_c / convergence）
  - first_detected: 初回検出日
  - current_state: 現在の状態
  - days_active: 継続日数
  - score_history: スコアの推移（JSON配列）
  - last_updated: 最終更新日
```
